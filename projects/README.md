# Projects

Runnable demos and reference implementations that complement the notebooks.

Suggested structure:
- `projects/backend/` (FastAPI apps, background workers, etc.)
- `projects/devops/` (Terraform modules, Docker/K8s examples, CI/CD pipelines)
- `projects/frontend/` (small UI demos, framework comparisons)
- `projects/data_science/` (pipelines, training/eval demos, MLOps sketches)
- `projects/cybersecurity/` (safe labs, toy apps for defensive demos)
